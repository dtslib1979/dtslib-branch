## 🔑 백오피스 들어가는 법

1. 위 백오피스 주소 클릭
2. 비밀번호: `1126` 또는 본인 지사 ID (예: `koosy`)
3. 4시간 동안 로그인 유지됨

---

## 📥 백오피스에서 볼 수 있는 것

### 1. INBOX — 본사 알림
- 박씨가 보낸 "이런 거 있어요" 메시지
- **명령 아님**. 참고만 하면 됨
- 배지 색상: `success`(녹색) / `info`(파랑) / `warning`(노랑)

### 2. OUTBOX — 내가 본사에 보낸 보고 (자동)
- 내가 뭐 올릴 때마다 자동으로 본사에 신호 감
- 격일 오전 9시에도 "저 살아있어요" 자동 보고
- 내가 직접 할 건 없음

### 3. SUBSCRIPTIONS — 내 구독
- 내 지사가 본사에서 받는 피드 목록
- `catalog.json` 에서 자동 동기화

### 4. MANIFEST — 내 지사 정체성
- `cellId`, `cellRepo`, `hqProtocolVersion` 같은 기본 정보

## ✍️ 내가 콘텐츠 만들고 싶을 때

**직접 git push 하려고 하지 마세요.**
박씨한테 다음 중 하나로 전달하면 됩니다:

1. **텔레그램**: 박씨 텔레그램에 내용/사진/링크 보냄
2. **카톡**: 박씨한테 카톡
3. **대면**: 직접 만나서 말로 설명
4. **이메일**: dimas@dtslib.com (선택)

박씨가 받아서 → 편집/정리 → 내 지사 레포에 올림 → 내 사이트에 표시됨.

---

## 🎤 내가 박씨한테 할 수 있는 요청

| 요청 | 예시 |
|---|---|
| 새 글 올려달라 | "이런 블로그 글 썼어요" |
| 사진/작품 전시 | "이 그림 올려주세요" |
| 사이트 디자인 바꿔달라 | "색깔 좀 더 밝게" |
| 새 기능 추가 | "구독 폼 만들어주세요" |
| 도메인 연결 | "내 도메인 사뒀어요, 연결해주세요" |

**→ 박씨가 "이런 거 있어요" 알림으로 다른 코워커들한테도 공유할 수 있음** (템플릿화)

## 📊 내 성장 과정 보기

내가 얼마나 했는지 보고 싶으면:

1. 백오피스 `OUTBOX` 섹션 — 최근 활동
2. 본사 원장 (박씨한테 부탁) — 시작부터 지금까지 전체
3. 내 지사 GitHub 페이지의 commit 이력 — 스토리 원본

**이게 다 당신의 성장 기록** 이고 박씨가 언젠가 콘텐츠화할 원자재입니다.
블로그/방송/책으로 나올 수 있어요.

## 🎓 졸업 — 자립할 때

이 조건이 충족되면 박씨가 "졸업" 제안할 수 있습니다:

- ✅ 본인 콘텐츠 아이디어를 꾸준히 냄
- ✅ GitHub 계정/도메인 운영법 이해
- ✅ 박씨 도움 없이 한 달 이상 운영 가능
- ✅ **본인이 원함**

졸업 = 내 지사 레포 소유권이 **내 GitHub 계정** 으로 이관됨.
→ 박씨 손 떠남. 독립 운영.

## ❓ 자주 묻는 질문

**Q. 내가 git 이나 기술 같은 거 몰라도 되나요?**
A. 네. 박씨가 다 합니다. 당신은 콘텐츠만.

**Q. 수익은 어떻게 나뉘나요?**
A. 박씨가 당신 성장기를 콘텐츠화해서 얻는 수익과, 당신 자체 활동 수익은 분리됩니다. 구체적인 건 박씨랑 직접 얘기하세요. 인큐베이터는 로열티 수거 구조 아닙니다.

**Q. 박씨가 내 허락 없이 뭘 올려요?**
A. 올리기 전에 당신한테 확인합니다. 공개된 것(블로그 글/사진)은 당신 제공 기반으로만 올라갑니다.

**Q. 언제든 그만둘 수 있나요?**
A. 네. 말씀만 하시면 레포 정리하고 안전하게 내려드립니다.

**Q. 백오피스 비번 까먹었어요.**
A. `1126` 또는 본인 지사 ID 소문자 (예: `koosy`). 박씨한테 물어봐도 됩니다.
